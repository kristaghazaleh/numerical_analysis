Introduction

This page collects the first main block of Math 128A into one place. The goal is to keep the key statements, formulas, proof patterns, and computational habits close together so the course reads as one connected subject instead of a pile of isolated techniques. Numerical analysis is mostly about three recurring questions: what the exact mathematics says, what finite precision changes, and how fast a method improves an approximation.

The material on this page centers on five themes. First, local calculus facts such as Taylor expansion, Rolle’s theorem, and the Mean Value Theorem are the tools used to justify numerical methods. Second, floating-point arithmetic explains why perfectly reasonable algebraic formulas can behave badly on a machine. Third, rootfinding methods compare robustness versus speed. Fourth, matrix factorizations explain how linear systems should actually be solved in practice. Fifth, interpolation builds explicit approximating polynomials and controls their error.

A practical way to read the whole subject is this: start with a mathematical target, replace it by a computable iteration or factorization, track what error is introduced at each stage, and then ask whether later steps shrink that error, preserve it, or magnify it. That same cycle appears in essentially every topic on this page. In rootfinding, the current iterate creates the next one and the key question is whether the new error is smaller than the old one. In linear systems, elimination creates roundoff and the key question is whether the matrix and algorithm amplify it. In interpolation, the data are exact only at the nodes and the key question is how the off-node error behaves.

Foundations

Calculus facts that keep showing up

Taylor expansion is the main local approximation tool. Around a point \(c\), \[ f(x)=\sum_{k=0}^{n} \frac{f^{(k)}(c)}{k!}(x-c)^k + E_n(x), \] and in the Lagrange form of the remainder, \[ E_n(x)=\frac{f^{(n+1)}(\xi)}{(n+1)!}(x-c)^{n+1} \] for some \(\xi\) between \(c\) and \(x\). The same statement can be written with \(h\) as \(f(x+h)=\sum_{k=0}^n f^{(k)}(x)h^k/k! + E_n(h)\). This is the cleanest way to explain truncation error, Newton’s method, and interpolation error.

Rolle’s theorem and the Mean Value Theorem are the standard bridge from exact roots to derivative information. They are what justify statements like “there is a point between these two values where the derivative has this value,” and that is exactly the kind of sentence that appears in convergence proofs.

Convergence language

If \(x_n \to x\), then the error is usually written as \(e_n=x_n-x\). A method has at least linear convergence if \(|e_{n+1}|\le C|e_n|\) for large \(n\) with some constant \(0<C<1\). It has superlinear convergence if \(|e_{n+1}|\le \omega_n |e_n|\) with \(\omega_n\to 0\). It has quadratic convergence if \(|e_{n+1}|\le C|e_n|^2\) for large \(n\). More generally, order \(\alpha\) means \(|e_{n+1}|\le C|e_n|^{\alpha}\).

The practical meaning is simple. Linear convergence removes error steadily but not dramatically. Quadratic convergence is the “digits start doubling” regime. Superlinear is in between. Sublinear means the sequence still converges, but the improvement per step is weak.

Big-Oh notation

Writing \(x_n = O(\omega_n)\) means there is a constant \(C>0\) such that \(|x_n|\le C|\omega_n|\) for all large \(n\). This is a boundedness statement, not a limit statement. It is often used to compress error formulas after a Taylor expansion.

How numerical problem solving usually works

Most problems in this course can be organized into four moves. First, identify the exact object: a root, a solution vector, or an interpolating polynomial. Second, choose a computable form: an iteration, a factorization, or a nested formula. Third, define the right error quantity. For roots this is usually \(e_n=x_n-r\); for linear systems it is often the solution error \(x-\tilde x\) together with the residual \(b-A\tilde x\); for interpolation it is \(f(x)-p(x)\). Fourth, compare one stage to the next. If the new error is about a constant times the old error, expect linear convergence. If the new error is about a constant times the square of the old error, expect quadratic convergence. If the new error is multiplied by something larger than 1, the process is unstable or divergent.

This is also how to talk through exam problems. Do not just compute. Say what the approximation is, what the exact target is, what quantity is being measured, and why the next step should improve or possibly worsen the approximation. A lot of numerical analysis questions are really asking whether you can explain the mechanism behind the numbers.

What to recognize immediately

  • If a proof uses local behavior near a root, expect Taylor expansion.
  • If a proof compares function values at two points, expect the Mean Value Theorem.
  • If a statement says many roots force a high derivative to vanish somewhere, expect Rolle’s theorem repeatedly.
  • If the question asks how fast a method converges, define \(e_n\) and try to relate \(e_{n+1}\) to \(e_n\).

Floating-Point Arithmetic, Error, and Stability

Machine representation

Real numbers are stored in normalized binary scientific notation. In floating-point form, a nonzero number looks like \(x=\pm q\,2^m\), where the mantissa is normalized. For Marc-32 the format is 1 sign bit, 8 exponent bits, and 23 mantissa bits. For IEEE 64-bit double the format is 1 sign bit, 11 exponent bits, and 52 mantissa bits. Most real numbers are not machine numbers, so the machine stores the nearest representable value instead.

Rounding and chopping

Chopping throws away all digits after a chosen place. Rounding replaces the number by the nearest available one. Rounding is usually more accurate. If \(\tilde x\) is a rounded \(n\)-digit approximation and \(\hat x\) is a chopped one, then rounding has a smaller worst-case error bound than chopping.

Absolute and relative error

If \(x\) is approximated by \(\tilde x\), then the absolute error is \(|x-\tilde x|\) and the relative error is \(|x-\tilde x|/|x|\), assuming \(x\neq 0\). Absolute error measures raw size of the mistake. Relative error measures scale-sensitive size. In numerical analysis, relative error is often the more meaningful quantity.

Machine epsilon

Machine epsilon measures the size of one-step relative rounding error. In Marc-32, the relative rounding error is bounded by about \(2^{-24}\). In IEEE 64-bit, the bound is much smaller. This is why double precision usually delays visible failure.

Catastrophic cancellation

The most dangerous arithmetic pattern is subtraction of nearly equal numbers. Even when each input is stored accurately, the leading digits cancel and the remaining digits can be mostly noise. Standard examples include the quadratic formula, expressions like \(\sqrt{x+1}-\sqrt{x}\), and \((\cos x - 1)/x^2\) near zero. The usual fix is algebraic reformulation, such as rationalizing or using an equivalent series-based form.

Stability and conditioning

Conditioning measures sensitivity of the mathematical problem itself. Stability measures whether the algorithm magnifies the small errors created during computation. A well-conditioned problem can still be computed badly by an unstable algorithm. A badly conditioned problem can defeat even a stable algorithm.

For a differentiable scalar function \(f\), a local condition number is often written as \[ \kappa_f(x)=\left|\frac{x f'(x)}{f(x)}\right|. \] Large magnitude means small relative changes in the input can cause large relative changes in the output.

Practical warnings

  • Never trust direct equality tests on floating-point numbers unless the problem is designed that way.
  • Reformulate before computing if subtraction of nearly equal terms is present.
  • Distinguish clearly between the exact formula and the floating-point implementation of that formula.
  • A recurrence can be mathematically correct and still be numerically unstable.

A good rule for error analysis is to ask two separate questions every time. First, how large is the data or rounding error entering this step? Second, what does the current formula do to that error? Some operations pass it along mildly, some shrink it, and some magnify it dramatically. That is why successive iterations matter so much in this course: even a tiny error can become dominant if the method keeps feeding it back through an amplifying mechanism.

Nonlinear Equations and Rootfinding

The general problem

The rootfinding problem is to solve \(f(r)=0\). Different methods make different tradeoffs. Bisection is slow but robust. Newton is usually fast but depends heavily on the initial guess and derivative information. Secant avoids explicit derivatives and is typically faster than bisection but less predictable than Newton. Fixed-point iteration gives a broader framework that explains why some schemes converge and others do not.

Bisection method

Start with an interval \([a_0,b_0]\) where \(f\) is continuous and \(f(a_0)f(b_0)<0\). Define the midpoint \(c_n=(a_n+b_n)/2\), keep the half interval where the sign change remains, and repeat. The method is guaranteed to converge to a root in the interval under the sign-change assumption.

The interval width halves every step: \[ b_n-a_n = 2^{-n}(b_0-a_0). \] The midpoint error satisfies \[ |r-c_n| \le 2^{-(n+1)}(b_0-a_0). \] This is why bisection is called linearly convergent: each step reduces the worst-case error bound by a factor of one half.

Strengths: guaranteed convergence, simple stopping rule, easy error bound. Weaknesses: slow, and it cannot find roots unless a sign change is available.

In successive iterations, the important quantity is not just the current midpoint but the interval that survives. Every step throws away half of the uncertainty and keeps the half that still contains the root. So the method is telling you exactly how ignorance shrinks: after one step, the uncertainty is half as large; after two steps, one quarter as large; after \(n\) steps, \(2^{-n}\) of the original width. This is why bisection is the cleanest method for error-analysis questions. It gives a guaranteed worst-case bound even if the actual root is behaving badly.

When a problem asks for the number of iterations, solve the inequality coming from the error bound. When it asks for the conceptual rate, say that each step multiplies the error bound by \(1/2\), so the convergence is linear. When it asks whether bisection is efficient, the honest answer is that it is efficient in reliability, not in speed.

Newton’s method

Newton’s update is \[ x_{n+1} = x_n - \frac{f(x_n)}{f'(x_n)}. \] Geometrically, this is the x-intercept of the tangent line at \((x_n,f(x_n))\). If \(r\) is a simple root and \(f\in C^2\), then for initial guesses sufficiently near \(r\), Newton’s method has quadratic convergence: \[ |x_{n+1}-r| \le C|x_n-r|^2. \]

Strengths: very fast near a simple root. Weaknesses: can diverge, can land where \(f'(x_n)=0\), and is only locally reliable. A good initial guess matters.

The practical error story is that Newton spends its early steps trying to enter the local regime where the tangent line is a good model of the function. Once it is close enough to a simple root, the dominant error term becomes quadratic, so the next error is roughly proportional to the square of the current error. That means if the current error is already small, the next one becomes dramatically smaller. This is the source of the familiar jump in correct digits after a few successful steps.

But the same tangent-line idea explains failure too. If the tangent is nearly horizontal, the correction term can explode. If the function is curved in an unfortunate way relative to the starting point, the tangent may send the iterate away from the root. If the root has multiplicity greater than one, the quadratic behavior is lost and the method often drops to linear-style behavior. So on conceptual questions, the right answer is usually not just “Newton is fast,” but “Newton is fast once the local Taylor model is accurate and the derivative at the root is nonzero.”

Secant method

The secant update is \[ x_{n+1}=x_n - f(x_n)\frac{x_n-x_{n-1}}{f(x_n)-f(x_{n-1})}. \] It replaces the derivative in Newton’s method by a finite-difference slope. That means no symbolic derivative is needed. For a simple root and enough smoothness, the secant method is superlinear with order \[ \frac{1+\sqrt{5}}{2}\approx 1.618. \] So it is faster than linear convergence, but slower than quadratic convergence.

Fixed-point iteration

Rewrite the equation \(f(r)=0\) as \(r=F(r)\). Then iterate \[ x_{n+1}=F(x_n). \] The point is not that every rearrangement works. The point is to choose a rearrangement that is contractive.

If \(F:[a,b]\to[a,b]\) and \(|F'(x)|\le \omega < 1\) on the interval, then \(F\) is a contraction. In that case there is a unique fixed point \(s\in[a,b]\), and for any starting value in the interval, \(x_n\to s\). The error estimate is \[ |s-x_n| \le \frac{\omega^n}{1-\omega}|x_1-x_0|. \]

Interpretation of \(|F'(s)|\): if it is bigger than 1, expect divergence; if it is near 1, expect slow convergence; if it is near 0, expect very fast local convergence. Newton’s method can itself be viewed as a fixed-point method, with derivative zero at a simple root, which is why it becomes quadratic.

This is one of the most useful practical frameworks in the course. If you can rewrite a method as \(x_{n+1}=F(x_n)\), then many conceptual questions become derivative questions. A graph crossing the line \(y=x\) with slope magnitude less than 1 tends to pull nearby iterates inward. A crossing with slope magnitude greater than 1 tends to push them outward. If the slope is positive, the approach is often monotone; if it is negative, oscillation is common. So when your exam asks what successive iterations will do, the sign and size of \(F'\) are often the main story.

Aitken acceleration

For linearly convergent fixed-point sequences, Aitken’s extrapolation can dramatically improve the approximation. One useful form is \[ s \approx x_n - \frac{d_n^2}{d_n-d_{n-1}}, \qquad d_n=x_n-x_{n-1}. \] This can be excellent for slowly convergent fixed-point iterations, but it is not automatically an improvement for every method. In particular, it is often unnecessary or even harmful when the base method is already quadratically convergent, as in Newton’s method.

Roots of polynomials and Horner’s method

Horner’s method evaluates a polynomial efficiently and stably in nested form. For \(p(x)=a_nx^n+\cdots+a_1x+a_0\), write \[ p(x)=(((a_nx+a_{n-1})x+a_{n-2})x+\cdots)x+a_0. \] A second Horner pass gives \(p'(x)\) at the same evaluation point. Complete Horner also produces coefficients of the expansion in powers of \((x-c)\), which is useful when a problem asks for \(p(x)=\sum_{k=0}^n a_k(x-c)^k\).

Linear Systems, Elimination, Factorizations, and Norms

Triangular systems

The systems that are actually easy to solve are triangular systems. For upper triangular systems use back substitution. For lower triangular systems use forward substitution. This is why elimination is valuable: it converts a hard system into triangular ones.

LU factorization

If a matrix admits a factorization \(A=LU\), where \(L\) is lower triangular and \(U\) is upper triangular, then solving \(Ax=b\) becomes two steps: first solve \(Ly=b\), then solve \(Ux=y\). The elimination multipliers populate \(L\), while the final upper matrix is \(U\).

Doolittle factorization is the version where the diagonal entries of \(L\) are all 1. It is convenient because it reduces the number of unknown factor entries.

Gaussian elimination

Gaussian elimination removes entries below the pivot one column at a time. The central structural requirement is that the pivots stay nonzero. If all required pivot entries remain nonzero, elimination yields an LU decomposition.

In exact arithmetic this is algebraically straightforward. In floating-point arithmetic it can be numerically dangerous, because tiny pivots lead to large multipliers, and large multipliers amplify rounding error.

Pivoting

Partial pivoting swaps rows so the pivot has largest absolute value in the current column. Scaled partial pivoting compares candidate pivots after normalizing by the largest entry in each row. The point is the same in both cases: avoid division by a small number when a much better pivot is available.

In practice, pivoting usually improves accuracy because it keeps multipliers smaller. A recurring course theme is that the same exact algebra can produce very different numerical results depending on whether pivoting is used.

For successive elimination steps, the danger is cumulative. Each row operation is performed in floating-point arithmetic, so each new row already carries tiny rounding errors from earlier steps. If a later step multiplies those errors by a large multiplier, they can become visible in the final answer. Pivoting helps because smaller multipliers mean less amplification from stage to stage. So when a question asks why elimination without pivoting can fail, the real answer is not just “small pivots are bad” but “small pivots create large multipliers, and large multipliers spread and amplify earlier roundoff through the remaining steps.”

PLU form

With row swaps included, the clean factorization is \[ PA = LU, \] where \(P\) is a permutation matrix. Determinants then satisfy \[ \det(P)\det(A)=\det(L)\det(U). \] Since triangular matrix determinants are products of diagonal entries, this gives a quick route to \(\det(A)\) after elimination.

Matrix norms and subordinate norms

A subordinate matrix norm is the one induced by a vector norm: \[ \|A\| = \max_{\|x\|=1} \|Ax\|. \] These norms satisfy the key inequalities \[ \|Ax\|\le \|A\|\,\|x\|, \qquad \|AB\|\le \|A\|\,\|B\|, \qquad \|I\|=1. \] For the infinity norm, \[ \|A\|_\infty = \max_i \sum_j |a_{ij}|, \] which is the largest absolute row sum.

Not every matrix norm is subordinate. If a question asks whether a given matrix expression defines a norm, check positivity, homogeneity, and the triangle inequality. If it asks whether it is subordinate, then there must also exist some vector norm so that the expression equals \(\sup_{\|x\|=1}\|Ax\|\).

Residual, error, and condition number

If \(\tilde x\) is an approximate solution of \(Ax=b\), then the error vector is \(e=x-\tilde x\) and the residual is \[ r=b-A\tilde x = A(x-\tilde x)=Ae. \] A small residual does not automatically mean a small error; the matrix may be ill-conditioned.

For an invertible matrix and a subordinate norm, the condition number is \[ \kappa(A)=\|A\|\,\|A^{-1}\|. \] Large \(\kappa(A)\) means the system is sensitive: small relative changes in data can produce large relative changes in the solution.

Iterative refinement

If \(\tilde x\) is approximate, compute the residual \(r=b-A\tilde x\), solve \(Ay=r\), and use \(\tilde x+y\) as a corrected solution. Since \[ A(\tilde x+y)=A\tilde x + Ay = A\tilde x + (b-A\tilde x)=b, \] the corrected vector can be much better, especially when the residual equation is solved carefully.

Sparse structure matters

The cost of solving a system depends strongly on matrix structure. A tridiagonal matrix can often be solved in linear time once its sparse factorization is used, instead of the cubic-style work associated with dense elimination. When a problem mentions bidiagonal, tridiagonal, or banded structure, pay attention to operation counts.

Polynomial Interpolation, Divided Differences, and Hermite Data

Polynomial interpolation

Given distinct nodes \(x_0,\dots,x_n\) and data \(y_j=f(x_j)\), there is a unique polynomial of degree at most \(n\) that matches all the data. Uniqueness comes from the fact that a nonzero polynomial of degree at most \(n\) cannot have \(n+1\) distinct roots.

Lagrange form

The interpolating polynomial can be written as \[ p(x)=\sum_{k=0}^n y_k \ell_k(x), \] where the cardinal functions are \[ \ell_k(x)=\prod_{j\ne k}\frac{x-x_j}{x_k-x_j}. \] Each \(\ell_k\) satisfies \(\ell_k(x_j)=\delta_{kj}\). Lagrange form is conceptually clean, but not always the best for repeated computation.

Newton form

Newton form builds the polynomial incrementally: \[ p_n(x)=c_0 + c_1(x-x_0) + c_2(x-x_0)(x-x_1)+\cdots + c_n\prod_{j=0}^{n-1}(x-x_j). \] This form is better for adding new nodes and for evaluation in nested form.

Divided differences

The Newton coefficients are divided differences. The first ones are \[ f[x_0]=f(x_0), \qquad f[x_0,x_1]=\frac{f(x_1)-f(x_0)}{x_1-x_0}. \] Higher-order divided differences satisfy the recursion \[ f[x_0,\dots,x_n]=\frac{f[x_1,\dots,x_n]-f[x_0,\dots,x_{n-1}]}{x_n-x_0}. \] The top row of the divided-difference table supplies the Newton coefficients directly.

Interpolation error

If \(f\in C^{n+1}\) and \(p\) is the degree-\(n\) interpolating polynomial at nodes \(x_0<\cdots<x_n\), then for each \(x\in[x_0,x_n]\), \[ f(x)-p(x)=\frac{f^{(n+1)}(\omega)}{(n+1)!}\prod_{j=0}^n (x-x_j) \] for some \(\omega\in(x_0,x_n)\). The error is naturally small near one of the nodes because one factor in the product is then small.

Increasing the degree is not always better. On a uniform grid, very high degree interpolation can behave badly. Node placement matters.

The practical viewpoint is that interpolation error comes from two sources at once: the smoothness of \(f\), through \(f^{(n+1)}\), and the geometry of the nodes, through the product \(\prod_{j=0}^n (x-x_j)\). So a problem about interpolation error is not just asking whether the function is nice. It is also asking where the evaluation point sits relative to the chosen nodes. Near a node one factor is small, so the error tends to be small. Far from the nodes, or with poorly spaced nodes, the product can become large and wipe out the benefit of higher degree.

Hermite interpolation

Hermite interpolation matches function values and derivative values. Given nodes \(x_0<\cdots<x_n\), values \(y_j\), and derivatives \(y'_j\), there is a unique polynomial of degree at most \(2n+1\) such that \[ p(x_j)=y_j, \qquad p'(x_j)=y'_j, \] for every \(j\).

In divided-difference language, repeated nodes encode derivative data. For example, \[ f[x_0,x_0]=f'(x_0). \] More generally, Hermite tables are built by repeating nodes and inserting derivative information into the repeated-node entries.

Problem Patterns and What They Usually Want

If the problem is about error or precision

  • Separate absolute error from relative error immediately.
  • Ask whether the issue is conditioning, stability, or both.
  • Look for cancellation and try an algebraic reformulation.
  • If a matrix is involved, check whether the condition number is the actual bottleneck.

If the problem is about convergence of an iteration

  • Define the fixed-point map \(F\) if possible.
  • Check whether \(|F'(x)|<1\) on an interval or at least near the fixed point.
  • For Newton or secant, identify whether the root is simple.
  • Write \(e_n=x_n-r\) and try to derive \(e_{n+1}\) in terms of \(e_n\) or \((e_n,e_{n-1})\).
  • Say explicitly what happens from one iteration to the next: halves, multiplies by a constant, squares, oscillates, or grows.
  • If the derivative at the fixed point is close to 1 or \(-1\), expect slow convergence or oscillation even if the method still converges.

If the problem is about bisection

  • The interval width after \(n\) steps is \(2^{-n}(b_0-a_0)\).
  • The midpoint error is at most half of that width.
  • Iteration-count questions usually come from solving the inequality for \(n\).

If the problem is about elimination or LU

  • Track the multipliers carefully; they become entries of \(L\).
  • Watch whether the problem wants \(A=LU\) or \(PA=LU\).
  • If numerical behavior is part of the question, discuss pivoting and condition number separately.
  • When asked why a computed solution is inaccurate, check three different causes: bad conditioning of \(A\), unstable elimination steps, or both.
  • Use residual language carefully: a small residual means the approximate vector nearly satisfies the equation, not automatically that it is close to the true solution.

If the problem is about interpolation

  • Write the node set and degree first.
  • Use Lagrange form for conceptual construction and Newton form for table-based computation.
  • If error is asked for, write the \((n+1)\)-st derivative term and the node product term.
  • Ask where the evaluation point sits relative to the nodes; this often explains the size of the error more clearly than the formula alone.
  • Do not confuse exact fit at the nodes with good behavior everywhere between or beyond the nodes.

How to Analyze Successive Iterations

The core habit

When a method produces a sequence \(x_0,x_1,x_2,\dots\), the main question is not just whether the numbers look closer. The real question is what rule carries one error into the next. Write the exact target as \(r\) or \(x\), define the current error, and then compare stages. For scalar rootfinding this usually means \(e_n=x_n-r\). For linear systems it may mean comparing a new residual or correction to the previous one. Once that relation is visible, the behavior of the whole method becomes much easier to predict.

In practice, look at three things at every step: the size of the update, the sign pattern of the update, and the ratio between successive errors or corrections whenever that ratio makes sense. These three observations usually tell you whether the method is contracting, oscillating, stalling, or blowing up.

What different iteration patterns usually mean

  • If \(|e_{n+1}|pprox c|e_n|\) with a nearly constant \(0<c<1\), expect linear convergence.
  • If \(|e_{n+1}|pprox c|e_n|^2\), expect quadratic convergence.
  • If \(|e_{n+1}|/|e_n| o 0\) but not quadratically, expect superlinear convergence.
  • If the sign of \(e_n\) alternates while the magnitude shrinks, the method is often converging with oscillation.
  • If the magnitude barely shrinks, the contraction factor is probably close to 1 or the method has entered a slow regime.
  • If updates suddenly stop improving, finite precision, cancellation, or a bad local model may have become dominant.
  • If the updates grow, the iteration is diverging or has left the region where the analysis applies.

Bisection across steps

Bisection is the cleanest case because the iteration acts on an interval rather than only on a point. The uncertainty interval is halved every step, so the worst-case midpoint error is halved every step as well. Nothing mysterious is happening from one stage to the next: the method simply keeps the half that still contains the root. If a problem asks what happens after many iterations, the correct answer is that uncertainty shrinks geometrically by a factor of \(1/2\) per step.

Newton across steps

Newton should be read in two phases. Before the iterate is close to the root, the tangent-line model may or may not help. After the iterate enters the local regime near a simple root, the first-order error term is cancelled and the dominant term is quadratic. That is why the method can look ordinary at first and then suddenly become extremely accurate. When an exam asks what happens in successive Newton iterates, mention both phases if needed: entry into the local regime and then quadratic error squaring.

A useful practical signal is the correction size \(\Delta_n = -f(x_n)/f'(x_n)\). If \(|\Delta_n|\) is shrinking rapidly and the iterates are stabilizing near a root, the local model is probably working. If \(|\Delta_n|\) becomes huge or erratic, a small derivative, bad starting point, or non-simple root may be the reason.

Secant across steps

The secant method uses two previous iterates, so its step-to-step behavior is naturally less regular than Newton’s. The slope is estimated from two points, which means the update can react strongly if those function values are very close or if the secant line is a poor local model. Conceptually, the method tends to improve faster than linear methods once it is behaving well, but the early iterates can be more uneven because the slope information is only approximate.

Fixed-point iteration across steps

For \(x_{n+1}=F(x_n)\), the derivative of \(F\) near the fixed point controls the story. If \(|F'(r)|<1\), nearby errors are multiplied by roughly \(|F'(r)|\), so they shrink. If \(F'(r)\) is negative, the iterates often alternate across the fixed point. If \(F'(r)\) is close to 0, convergence can be very fast. If \(|F'(r)|>1\), nearby errors are pushed outward and the iteration is locally repelling.

This gives a strong template for conceptual answers: say whether the map pulls points inward or pushes them outward, and explain whether the sign of the derivative suggests monotone approach or oscillation. Many “describe the iteration” questions are really asking for that interpretation.

Successive elimination steps and propagated error

Elimination is not usually written as \(x_{n+1}=F(x_n)\), but it is still a multistage process where each step feeds into the next. Each row operation creates new rounded entries. Later row operations use those rounded entries again. So the key conceptual question is whether the algorithm keeps these local errors controlled or multiplies them through large elimination factors. That is why pivoting is so important: it changes the later stages by changing the size of the multipliers that act on existing error.

Residual-based correction across steps

In iterative refinement, the current approximate solution creates a residual, the residual creates a correction, and the correction produces the next approximate solution. The method works when the residual equation is solved accurately enough that each new correction removes a meaningful portion of the current error. So if a question asks what successive refinement steps are doing, the answer is that each step tries to solve for the error left over from the previous one.

What to say when the problem asks for a verbal analysis

  • Name the exact target and define the error quantity.
  • State what one iteration does to that error: halves it, multiplies it by a constant, squares it, alternates it, or amplifies it.
  • Say whether the behavior is local or global. Newton conclusions are usually local; bisection conclusions are global on the interval.
  • Distinguish true mathematical convergence from finite-precision stalling.
  • If the method is unstable, say what mechanism causes the instability, not just that the answer is bad.

A compact checklist

For any iterative question, ask: what is the target, what is the current error, what formula generates the next step, what does that formula do to the current error, and what pattern should appear after many repetitions? If you answer those five things clearly, you usually have the full conceptual analysis the problem wants.

Questions and Answers

What is the difference between conditioning and stability?

Conditioning belongs to the mathematical problem. It asks whether small changes in input produce small or large changes in the exact answer. Stability belongs to the algorithm. It asks whether roundoff and local computational errors stay controlled or get amplified.

Why can a tiny residual still be misleading?

Because \(r=b-A\tilde x = Ae\). If \(A^{-1}\) is large in norm, then a small residual can correspond to a much larger solution error. That is exactly why condition numbers matter.

Why is bisection so dependable?

It uses only continuity and a sign change. Every step preserves an interval that still contains a root, and the interval length halves each time. The method is slow, but it is very hard to break when its assumptions are satisfied.

Why is Newton so fast near a simple root?

Because the tangent-line correction removes the first-order error term. What remains is second-order, which is why the new error is proportional to the square of the old one.

Why can Newton fail?

The derivative may vanish, the initial guess may not be in a region where the method is attracted to the desired root, or the tangent step may jump into a bad region. Fast local behavior does not guarantee good global behavior.

Why does pivoting help?

It avoids dividing by small pivots when a larger pivot is available. That keeps the elimination multipliers smaller, which usually reduces roundoff growth.

What does a divided difference actually represent?

It is the next Newton coefficient. The highest-order divided difference is the leading coefficient of the interpolating polynomial over the listed nodes.

Why is direct polynomial form often a bad way to evaluate interpolation formulas?

Because direct expansion is expensive and can be numerically awkward. Newton form plus nested evaluation is usually cleaner computationally.

What should I do when a formula contains subtraction of nearly equal terms?

Stop and reformulate before plugging numbers in. Rationalize, factor, use an equivalent identity, or switch to a series expansion if the point is local behavior.

What should I watch in successive iterations?

Watch the size, sign, and pattern of the updates. If the error is being multiplied by an approximately constant factor less than 1, that is linear behavior. If the number of correct digits starts increasing much faster, you are likely seeing quadratic or superlinear behavior. If the iterates alternate around the target, the derivative of the fixed-point map may be negative. If the updates stop shrinking, the method may have stalled, reached finite-precision limits, or left the local convergence regime.

Why do so many questions separate residual from error?

Because they measure different things. Residual asks how well the computed answer satisfies the original equation. Error asks how far the computed answer is from the exact one. In a well-conditioned problem they often track each other well. In an ill-conditioned problem they can be very different, so you have to discuss both.

What is the main practical difference between exact arithmetic and numerical analysis?

Exact arithmetic only cares whether the algebra is correct. Numerical analysis also cares whether each stage is safe under finite precision. A derivation can be perfectly correct and still be a poor computational method if it introduces cancellation, uses unstable recurrences, or magnifies data error too strongly.

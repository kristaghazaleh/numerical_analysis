Introduction

Numerical analysis studies how exact mathematical problems are turned into computable procedures, what kinds of error are introduced in that process, and whether those errors shrink or grow as the computation continues. The subject is never just about getting a number. It is about understanding why a method works, what assumptions it needs, and how finite precision changes the picture.

Nearly every topic on this page can be read through the same chain of questions. What is the exact target? What approximation is being produced? What quantity measures the discrepancy? What theorem controls that discrepancy? What happens to the discrepancy after one more step? That basic cycle is what ties together convergence, floating-point error, rootfinding, elimination, conditioning, and interpolation.

In this course, theorems matter because they tell you when a method is valid, not just how to run it. Definitions matter because many exam questions are really testing whether you can identify the right object: sequence, limit, relative error, condition number, simple root, subordinate norm, residual, divided difference, interpolation error, and so on. The page below is organized around those objects first and the algorithms second.

Foundations

Core definitions

A sequence is a function from the positive integers into the real numbers, usually written as \((x_n)=(x_1,x_2,x_3,\dots)\). A sequence converges to \(x\) if for every \(\varepsilon>0\) there exists an integer \(N\) such that whenever \(n>N\), we have \(|x_n-x|<\varepsilon\). This is the full epsilon-definition of limit, and it is the foundation under every later convergence claim.

The error in an approximation \(x_n\) to an exact target \(x\) is often denoted by \(e_n=x_n-x\). In rootfinding problems the exact target is typically a root \(r\), so \(e_n=x_n-r\). In many proofs the sequence itself is less important than the recurrence satisfied by the errors.

A root \(r\) of \(f\) is called a simple root if \(f(r)=0\) and \(f'(r)\neq 0\). Simple roots matter because Newton and secant error theorems are typically stated for them. A root of multiplicity greater than one behaves differently and often destroys the faster convergence rate.

Convergence-rate definitions

A sequence \((x_n)\) converging to \(x\) has linear convergence if there is a constant \(C\) with \(0<C<1\) such that for all sufficiently large \(n\), \[ |x_{n+1}-x|\le C|x_n-x|. \] It has superlinear convergence if there are numbers \(\omega_n\to 0\) with \[ |x_{n+1}-x|\le \omega_n |x_n-x| \] for all large \(n\). It has quadratic convergence if there is a constant \(C>0\) such that \[ |x_{n+1}-x|\le C|x_n-x|^2 \] for all large \(n\). More generally, convergence has order \(\alpha>1\) if the next error behaves like a constant times the current error raised to the power \(\alpha\).

Practically, linear convergence means the error shrinks by roughly a fixed factor each step. Quadratic convergence means the method enters a regime where the number of correct digits typically grows very rapidly because the new error is approximately proportional to the square of the old one.

Big-Oh language

Writing \(x_n = O(\omega_n)\) means there exists a constant \(C>0\) and an index \(N\) such that \(|x_n|\le C|\omega_n|\) whenever \(n>N\). This is a boundedness statement used to compress asymptotic formulas, especially after Taylor expansion.

The calculus facts used everywhere

The Mean Value Theorem says that if \(f\) is continuous on \([a,b]\) and differentiable on \((a,b)\), then there exists \(\xi\in(a,b)\) such that \[ f'(\xi)=\frac{f(b)-f(a)}{b-a}. \] In numerical analysis this theorem is the bridge between differences of function values and derivatives. It is the tool used to prove uniqueness of fixed points, derive local condition-number formulas, and compare consecutive iterates.

Rolle's theorem says that if \(f\) is continuous on \([a,b]\), differentiable on \((a,b)\), and \(f(a)=f(b)\), then there exists \(\xi\in(a,b)\) with \(f'(\xi)=0\). In interpolation theory it is repeatedly applied to prove uniqueness and error formulas.

The Taylor formula with remainder says that if \(f\in C^{n+1}\) near \(c\), then \[ f(x)=\sum_{k=0}^n \frac{f^{(k)}(c)}{k!}(x-c)^k + \frac{f^{(n+1)}(\xi)}{(n+1)!}(x-c)^{n+1} \] for some \(\xi\) between \(c\) and \(x\). This is the main source of truncation-error formulas, Newton error analysis, and interpolation remainder terms.

How to use these ideas on problems

  • If the question asks how fast something converges, define an error \(e_n\) immediately.
  • If the question compares function values at nearby points, expect the Mean Value Theorem.
  • If the question asks for a local error formula or rate, expect Taylor expansion.
  • If the question claims too many roots or interpolation conditions, expect Rolle's theorem or repeated differentiation logic.
  • If the problem describes successive approximations verbally, translate the story into a relation between \(e_{n+1}\) and \(e_n\).

Floating-Point Arithmetic, Error, and Stability

Basic definitions

If an exact quantity \(x\) is approximated by \(\tilde x\), the absolute error is \(|x-\tilde x|\) and the relative error is \(|x-\tilde x|/|x|\) when \(x\neq 0\). Absolute error measures raw size of the discrepancy. Relative error measures discrepancy relative to the scale of the true quantity and is often the more meaningful quantity in computation.

A numerical process is called unstable if small errors introduced at one stage are magnified in later stages so that the final result loses significant accuracy. This is an algorithmic notion. It is different from conditioning, which measures sensitivity of the mathematical problem itself.

For a differentiable scalar function \(f\), a local condition number at \(x\) is \[ \kappa_f(x)=\left|\frac{x f'(x)}{f(x)}\right|, \] provided \(f(x)\neq 0\). Large magnitude means small relative changes in the input can produce large relative changes in the output.

Floating-point representation

A computer stores only finitely many real numbers exactly. A nonzero floating-point number is stored in normalized scientific notation, with a sign, an exponent, and a mantissa. Most real numbers are therefore replaced by nearby machine numbers. This first approximation is usually called rounding or chopping, depending on the rule used.

Chopping discards digits beyond the storage limit. Rounding replaces the number by the nearest representable machine number. Rounding usually gives the smaller worst-case error bound. The smallest significant relative spacing of nearby machine numbers is controlled by the machine epsilon, which serves as a model size for one-step relative rounding error.

Error propagation and cancellation

The most dangerous arithmetic pattern is subtraction of nearly equal numbers. This is catastrophic cancellation. When leading digits cancel, the remaining digits may reflect earlier rounding noise more than genuine information. Standard examples include the smaller root in the quadratic formula, \(\sqrt{x+1}-\sqrt{x}\) for large \(x\), and formulas such as \((\cos x-1)/x^2\) near zero.

The fix is usually not to compute more carelessly, but to compute an algebraically equivalent expression that avoids cancellation. Rationalizing a numerator, switching to a conjugate form, or using a Taylor approximation are all examples of stability-improving reformulation.

Theoretical viewpoint on conditioning

If the input \(x\) is perturbed to \(x+h\), then by Taylor expansion, \[ f(x+h)-f(x)=f'(x)h+O(h^2). \] Dividing by \(f(x)\) shows that the relative output change is locally controlled by \(\kappa_f(x)\) times the relative input change. This is why the derivative-based condition number appears naturally and why large derivative relative to function size signals sensitivity.

What exam-style questions often want here

  • State whether the issue is conditioning of the problem or stability of the algorithm, and do not confuse them.
  • Compute both absolute and relative error if asked, then explain which one is more informative.
  • Identify cancellation before doing arithmetic.
  • Explain how an apparently correct formula can still be numerically dangerous because finite precision feeds through later stages.

Nonlinear Equations and Rootfinding

General setting

The basic problem is to find \(r\) such that \(f(r)=0\). A numerical method produces approximations \(x_0,x_1,x_2,\dots\). The analysis then asks whether \(x_n\to r\), under what hypotheses this happens, and how the errors \(e_n=x_n-r\) behave.

Bisection method

The method begins with an interval \([a_0,b_0]\) on which \(f\) is continuous and \(f(a_0)f(b_0)<0\). By the Intermediate Value Theorem, there exists a root \(r\in(a_0,b_0)\). Let \[ c_n=\frac{a_n+b_n}{2}. \] Choose the half-interval where the sign change persists and repeat.

The key theorem-level fact is that the interval width halves each time: \[ b_n-a_n = 2^{-n}(b_0-a_0). \] Since the root remains in \([a_n,b_n]\), the midpoint error satisfies \[ |r-c_n|\le \frac{b_n-a_n}{2}=2^{-(n+1)}(b_0-a_0). \] This is a fully explicit error bound and is the standard reason bisection is classified as linearly convergent.

In conceptual terms, bisection does not guess a better point directly. It shrinks uncertainty geometrically. That makes it slow but extremely reliable. If a question asks how many iterations are needed for a certain tolerance, solve the midpoint-error inequality for \(n\).

Newton's method

Newton's iteration is \[ x_{n+1}=x_n-\frac{f(x_n)}{f'(x_n)}. \] Geometrically, \(x_{n+1}\) is the x-intercept of the tangent line to \(f\) at \((x_n,f(x_n))\).

The main theorem used in this course is the local error theorem: if \(f\in C^2\), \(r\) is a simple root, and \(x_n\) remains sufficiently close to \(r\), then there is a constant \(C>0\) such that \[ |x_{n+1}-r|\le C|x_n-r|^2. \] This is the precise statement of quadratic convergence. It comes from Taylor expanding \(f\) near the root and using the fact that the first-order term is cancelled by the Newton correction.

The hypotheses matter. The derivative must not vanish at the root, the function must be smooth enough, and the initial guess must enter the local basin where the tangent-line model is accurate. If \(f'(x_n)=0\) or the iterates wander away, the method can fail despite being very fast near a good root.

Secant method

The secant method replaces the derivative in Newton's formula by a finite-difference slope: \[ x_{n+1}=x_n-f(x_n)\frac{x_n-x_{n-1}}{f(x_n)-f(x_{n-1})}. \] It uses two previous iterates instead of an exact derivative. For a simple root under suitable smoothness, the convergence is superlinear, with order \[ \alpha=\frac{1+\sqrt5}{2}. \] So it is faster than linear convergence but slower than quadratic convergence.

Conceptually, the secant method inherits Newton's local philosophy but with approximate slope information. That is why it can be faster than bisection without needing symbolic derivatives, yet more erratic than Newton in early steps.

Fixed-point iteration

A fixed point of a function \(F\) is a number \(s\) such that \(F(s)=s\). To solve \(f(r)=0\), one often rewrites the equation as \(x=F(x)\) and iterates \[ x_{n+1}=F(x_n). \]

Two theorem-level facts are central here. First, if \(F:(0,1)\to(0,1)\) is differentiable and \(|F'(x)|<1\) throughout the interval, then there can be at most one fixed point; this follows from the Mean Value Theorem. Second, if \(F:[a,b]\to[a,b]\) is continuous, then it has at least one fixed point; this follows by applying the Intermediate Value Theorem to \(g(x)=F(x)-x\).

The strongest practical theorem is the contraction statement: if \(F:[a,b]\to[a,b]\) and \(|F'(x)|\le \omega<1\) for all \(x\in[a,b]\), then there is a unique fixed point \(s\in[a,b]\), and every iteration starting in \([a,b]\) converges to \(s\). Moreover, \[ |s-x_n|\le \frac{\omega^n}{1-\omega}|x_1-x_0|. \]

This is where many conceptual exam questions come from. If \(|F'(s)|>1\), nearby errors are pushed away. If \(|F'(s)|\approx 1\), convergence is slow. If \(F'(s)<0\), oscillation is common. If \(F'(s)=0\), the method can converge much faster locally.

Roots of polynomials and Horner's method

Horner's method rewrites a polynomial \[ p(x)=a_nx^n+a_{n-1}x^{n-1}+\cdots+a_1x+a_0 \] in nested form, \[ p(x)=(((a_nx+a_{n-1})x+a_{n-2})x+\cdots)x+a_0. \] This reduces arithmetic cost and is usually the preferred way to evaluate polynomials numerically. A related repeated process also gives coefficients in powers of \((x-c)\), which is useful when a problem asks for a shifted expansion.

Linear Systems, Elimination, Factorizations, and Norms

Definitions from matrix algebra

An elementary row operation is one of three actions on a matrix: interchange two rows, multiply a row by a nonzero scalar, or add a scalar multiple of one row to another row. An elementary matrix is a matrix obtained by applying one elementary row operation to the identity matrix. Left-multiplying by that elementary matrix performs the same row operation on any compatible matrix.

A matrix \(L\) is lower triangular if entries above the diagonal are zero. A matrix \(U\) is upper triangular if entries below the diagonal are zero. These systems are easy to solve by forward or backward substitution.

An LU factorization of a square matrix \(A\) is a representation \(A=LU\) with \(L\) lower triangular and \(U\) upper triangular. With row interchanges included, the more general factorization is \(PA=LU\), where \(P\) is a permutation matrix.

Invertibility theorem

A central theorem states that for a square matrix \(A\), the following are equivalent: \(A^{-1}\) exists, \(\det(A)\neq 0\), the system \(Ax=b\) has a unique solution for every \(b\), the homogeneous system \(Ax=0\) has only the trivial solution, and \(A\) can be written as a product of elementary matrices. This theorem explains why questions about uniqueness, determinants, invertibility, and elimination are all connected.

Gaussian elimination and pivoting

Gaussian elimination zeroes entries below a pivot one column at a time. In exact arithmetic it transforms the system into an equivalent upper triangular system. In floating-point arithmetic the danger comes from small pivots, because dividing by a tiny pivot produces large multipliers, and large multipliers amplify existing roundoff.

Partial pivoting chooses the largest available entry in the current column as the pivot. Scaled partial pivoting compares candidates after normalizing by row size. Both are attempts to control the growth of multipliers and therefore control numerical instability through the elimination stages.

Vector norms and matrix norms

A norm on a vector space is a function \(\|\cdot\|\) satisfying positivity, homogeneity, and the triangle inequality. On \(\mathbb R^n\), the standard examples are \[ \|x\|_1=\sum_{i=1}^n |x_i|,\qquad \|x\|_2=\left(\sum_{i=1}^n x_i^2\right)^{1/2},\qquad \|x\|_\infty=\max_i |x_i|. \]

Given a vector norm, the associated subordinate matrix norm is \[ \|A\|=\max_{\|u\|=1}\|Au\|. \] A theorem from the notes says this really is a norm on matrices. The important consequences are \[ \|Ax\|\le \|A\|\,\|x\|,\qquad \|AB\|\le \|A\|\,\|B\|,\qquad \|I\|=1. \] These are the workhorses of error bounds in linear systems.

The spectral norm or matrix 2-norm is \[ \|A\|_2=\max_{\|x\|_2=1}\|Ax\|_2. \]

Residual, error, condition number, and refinement

If \(\tilde x\) is an approximate solution to \(Ax=b\), then the error vector is \(e=x-\tilde x\) and the residual is \[ r=b-A\tilde x. \] Since \(r=Ae\), a small residual does not necessarily imply a small error unless the inverse of \(A\) is well controlled.

For an invertible matrix, the condition number in a chosen subordinate norm is \[ \kappa(A)=\|A\|\,\|A^{-1}\|. \] Large condition number means that the problem \(Ax=b\) is sensitive to perturbations in the data. This is a property of the matrix problem itself, not of any specific elimination algorithm.

Iterative refinement starts from \(\tilde x\), forms the residual \(r=b-A\tilde x\), solves \(Ay=r\), and uses \(\tilde x+y\) as a corrected approximation. The point is that the residual equation tries to solve directly for the remaining error.

Polynomial Interpolation, Divided Differences, and Hermite Data

Interpolation problem and uniqueness theorem

Given distinct nodes \(x_0,x_1,\dots,x_n\) and prescribed values \(y_0,y_1,\dots,y_n\), the interpolation problem asks for a polynomial \(p\) with degree at most \(n\) such that \[ p(x_j)=y_j\qquad (j=0,1,\dots,n). \] The key theorem says that if \(x_0<x_1<\cdots<x_n\), then for any real data \(y_0,\dots,y_n\), there exists a unique polynomial of degree at most \(n\) satisfying these conditions.

The uniqueness proof is important conceptually: if two such polynomials existed, their difference would be a polynomial of degree at most \(n\) with \(n+1\) distinct roots, so it would have to be the zero polynomial.

Lagrange form

The Lagrange basis polynomials are \[ \ell_k(x)=\prod_{j\ne k}\frac{x-x_j}{x_k-x_j}. \] They satisfy \(\ell_k(x_j)=\delta_{kj}\), where \(\delta_{kj}\) is the Kronecker delta. The interpolating polynomial is then \[ p(x)=\sum_{k=0}^n y_k\,\ell_k(x). \] This form is conceptually transparent because each basis function isolates one data value.

Newton form and divided differences

The Newton form is \[ p_n(x)=c_0+c_1(x-x_0)+c_2(x-x_0)(x-x_1)+\cdots+c_n\prod_{j=0}^{n-1}(x-x_j). \] The coefficients \(c_k\) are the divided differences.

They are defined recursively by \[ f[x_0]=f(x_0),\qquad f[x_0,x_1]=\frac{f(x_1)-f(x_0)}{x_1-x_0}, \] and for higher order, \[ f[x_0,\dots,x_n]=\frac{f[x_1,\dots,x_n]-f[x_0,\dots,x_{n-1}]}{x_n-x_0}. \] The highest-order divided difference over the listed nodes is exactly the leading Newton coefficient.

Interpolation error theorem

If \(f\in C^{n+1}\) on an interval containing the nodes and \(p\) is the degree-\(n\) interpolating polynomial, then for each \(x\) in the interval there exists \(\omega\) in the interval such that \[ f(x)-p(x)=\frac{f^{(n+1)}(\omega)}{(n+1)!}\prod_{j=0}^n (x-x_j). \] This formula shows that interpolation error depends on two things simultaneously: smoothness of the function through \(f^{(n+1)}\), and geometry of the nodes through the product term.

Hermite interpolation

Hermite interpolation prescribes both function values and derivative values. Given nodes \(x_0<\cdots<x_n\), function values \(y_j\), and derivative values \(y'_j\), the problem is to find a polynomial \(p\) with \[ p(x_j)=y_j,\qquad p'(x_j)=y'_j\qquad (j=0,1,\dots,n). \] The theorem from the notes says there exists a unique polynomial of degree at most \(2n+1\) satisfying these conditions.

Repeated nodes encode derivative data in divided-difference language. The first repeated-node divided difference is defined by \[ f[x_0,x_0]=f'(x_0). \] This is the basic bridge from ordinary divided differences to Hermite tables.

Problem Patterns

When the problem asks for a theorem

  • For existence of a root in an interval: continuity plus sign change gives the Intermediate Value Theorem.
  • For uniqueness of a fixed point: use the Mean Value Theorem and the bound \(|F'|<1\).
  • For local Newton rate: state the simple-root and \(C^2\) hypotheses before quoting quadratic convergence.
  • For interpolation existence and uniqueness: distinct nodes and degree at most \(n\).
  • For Hermite interpolation: values and first derivatives at \(n+1\) nodes give degree at most \(2n+1\).

When the problem is really about error analysis

  • Identify the exact target and define the error quantity before doing algebra.
  • Say whether you are tracking absolute error, relative error, residual, or interpolation remainder.
  • Explain what one more iteration does to the current error.
  • Separate finite-precision effects from exact-arithmetic convergence claims.

How to Analyze Successive Iterations

The general strategy

Successive-iteration questions are asking for a dynamical picture of the error, not just a list of computed numbers. Start by naming the target \(r\) or \(x\). Define the current error. Then identify the rule that maps one step to the next. If you can explain what the update does to the current error, you usually have the whole conceptual answer.

For bisection the update halves uncertainty. For a contraction mapping the update multiplies error by about \(|F'(r)|\). For Newton near a simple root the update makes the next error behave like a constant times the square of the current one. For elimination, each stage may propagate earlier rounding through new multipliers. In every case the key phrase is the same: what happens to the previous error after one more application of the algorithm?

Patterns to recognize

  • If \(|e_{n+1}|\approx c|e_n|\) with \(0<c<1\), the behavior is linear.
  • If \(|e_{n+1}|\approx c|e_n|^2\), the behavior is quadratic.
  • If the sign alternates while magnitude shrinks, the method is converging with oscillation.
  • If the ratio \(|e_{n+1}|/|e_n|\) stays close to 1, convergence is slow or stalled.
  • If updates grow, the method has left the region of attraction or the process is unstable.

What to say on written solutions

  • State whether the theorem being used is local or global.
  • Explain the mechanism of improvement, not just the final rate word.
  • Distinguish exact-arithmetic theory from finite-precision behavior if both matter.
  • When relevant, explain why roundoff or cancellation will eventually dominate later steps.

Questions and Answers

Why is a small residual not the same as a small error?

Because the residual is \(r=b-A\tilde x=A(x-\tilde x)=Ae\). If \(A^{-1}\) has large norm, then even a small residual can correspond to a much larger true solution error.

Why does Newton become so fast near a good root?

Because for a simple root the Newton correction cancels the first-order term in the Taylor expansion, leaving the dominant error term quadratic.

What is the practical meaning of a condition number?

It measures how much relative input error can be amplified by the mathematical problem itself. Large condition number means the exact answer is sensitive before any algorithm is even chosen.

Why do interpolation theorems always emphasize distinct nodes?

Distinct nodes prevent collapse of the interpolation conditions. If repeated information is desired, derivative data must be added explicitly, which is what Hermite interpolation does.

Why are theorem hypotheses so important in numerical analysis?

Because a method formula can still be algebraically correct when the theorem guaranteeing its behavior does not apply. Numerical analysis is full of methods that are excellent under the right hypotheses and unreliable outside them.
